# data/

Shared dataset and seed-data directory. Only small runtime artifacts live here.
Large raw datasets and intermediate LeetCode cleanup files are intentionally not
committed.

## Contents

| Path | What | Source |
|---|---|---|
| `fish_species.json` | Backend-readable catalog with 50 fish species. Each record includes name, species, rarity, catch probability, image path, description, sell value, and marketplace eligibility. | Generated by `scripts/build_fish_catalog.py` |
| `fish_images/<species_id>.png` | Committed AI-generated transparent PNG fish image assets. This is the only committed fish image directory. | Generated once with AI image tooling; verified by `scripts/build_fish_catalog.py` |
| `judgeable_problems.json` | Runtime coding-problem dataset used by game-service. Contains the 74 executable LeetCode-style problems with starter code, tests, and revealable answers. | Curated CatCh dataset |

## Regenerating

```bash
python3 scripts/build_fish_catalog.py
```

Re-running the catalog builder verifies `data/fish_images/*.png`, rewrites
`data/fish_species.json`, and keeps image URLs aligned with the backend static
route `/fish_images/<species_id>.png`.

## Why not commit the raw dataset

Raw LeetCode exports and intermediate cleanup files are not needed at runtime.
Keeping only the curated 74-problem JSON avoids confusing old 9/100-problem
datasets and keeps CI inputs deterministic.
